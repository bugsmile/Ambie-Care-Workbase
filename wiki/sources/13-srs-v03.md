# SRS v3.0 — 소프트웨어 요구사항 명세서

> **문서 ID:** SRS-001 v3.0 | **날짜:** 2026-04-20 | **표준:** ISO/IEC/IEEE 29148:2018  
> **기술 스택:** Next.js (App Router) + Prisma + Supabase Free + Vercel AI SDK + Vercel Hobby  
> **개발 방식:** 바이브 코딩 (AI 코딩 어시스턴트 기반, 1인 개발, SW 경험 3개월)

---

## 문서 버전 연혁 핵심

| 버전 | 날짜 | 핵심 변경 |
|-----|------|---------|
| v1.0 | 2026-04-18 | AWS 클라우드 기반 초기 SRS |
| v2.0 | 2026-04-19 | Next.js App Router + Supabase + Vercel 스택 전환 |
| **v3.0** | 2026-04-20 | **바이브 코딩 무료 티어 최적화** — 4개 주요 결정 반영 |

### v3.0 4대 핵심 결정

| 결정 | 변경 전 | 변경 후 | 이유 |
|-----|--------|--------|------|
| 실시간 푸시 | FCM/Web Push | **이메일 (Resend Free)** | 무료 제약 |
| SLA | 99.99% | **Best Effort ~99%** | Hobby 플랜 SLA 없음 |
| 데이터 보존 | 90일 | **30일** | Supabase Free 500MB 한계 |
| EMR 연동 | MVP Must | **Wave 2 연기** | DEP-01 미체결 + HMAC 전문가 필요 |

## MVP 범위 (3-Tier)

| 티어 | 범위 | 기간 | 인프라 비용 |
|-----|------|------|-----------|
| **MVP Core (Phase 0–3)** | 웹 대시보드 + 웰니스 리포트 + AI 요약 + 이메일 알림 + 모의 데이터 | 5–7주 | **$0/월** |
| **Phase 2 확장** | 수면 트렌드 차트 + PWA + Amplitude | +1–2주 | $0 유지 |
| **Wave 2 (별도)** | Edge AI/HW + EMR + SMS/카카오 + PagerDuty + 콜드 아카이빙 | 별도 | Vercel Pro + Supabase Pro |

### In-Scope (MVP Core)

| 기능 | Phase | 설명 |
|-----|-------|------|
| 모의 데이터 생성기 | **Phase 0** | Edge 센서 없는 개발/데모용 Seed 스크립트 |
| B2C 보호자 포털 (웹) | **Phase 1** | Next.js 기반 일일 리포트, 오탐 신고 |
| B2B 모니터링 대시보드 | **Phase 1** | 신호등 UI, API 폴링 30초 (Realtime은 Phase 2) |
| 웰니스 일일 리포트 | **Phase 2** | Vercel Cron 1회/일, Gemini AI 요약 포함 |
| 이메일 알림 (Resend Free) | **Phase 2** | MVP 알림 채널. 일 100건 제한 |

### Out-of-Scope (Wave 2 이관)

| 항목 | 이관 이유 |
|-----|---------|
| EMR Webhook (HMAC-SHA256) | DEP-01 미체결 + 전문 엔지니어 필요 |
| SMS/카카오톡 | 유료 (전액 무료 원칙 위배) |
| FCM/Web Push | Wave 2 (현재 Resend 이메일 대체) |
| 콜드 아카이빙 (90일 초과) | MVP 런칭 후 90일까지 불필요 |
| PagerDuty | 유료 ($21/user/월) |

## 기술 아키텍처

### 기술 스택 제약 (C-TEC)

| 제약 | 내용 |
|-----|------|
| C-TEC-001 | Next.js App Router 풀스택 단일 코드베이스 |
| C-TEC-002 | 모든 서버 로직 = Server Actions + Route Handlers |
| C-TEC-003 | Prisma ORM + 로컬 SQLite / 프로덕션 Supabase PostgreSQL |
| C-TEC-004 | Tailwind CSS + shadcn/ui |
| C-TEC-005 | Vercel AI SDK (`@ai-sdk/google`) |
| C-TEC-006 | Google Gemini 1.5 Flash (기본, env 변수로 교체 가능) |
| C-TEC-007 | Vercel Hobby 배포, Cron 1회/일 |

### Route Handlers (6개 MVP Core)

| # | 경로 | Method | 기능 | Phase |
|---|-----|--------|------|-------|
| 1 | `app/api/events/ingest` | POST | Mock 이벤트 주입 | Phase 0 |
| 2 | `app/api/reports/daily/[deviceId]/[date]` | GET | 일일 리포트 조회 | Phase 1 |
| 3 | `app/api/events/[eventId]/false-alarm` | POST | 오탐 신고 접수 | Phase 2 |
| 4 | `app/api/ai/wellness-summary` | POST | Gemini AI 요약 생성 | Phase 2 |
| 5 | `app/api/dashboard/status` | GET | B2B 신호등 상태 | Phase 1 |
| 6 | `app/api/devices/[deviceId]/heartbeat` | POST | 기기 생존 신호 | Phase 1 |

> Wave 2 이관: `app/api/webhooks/emr/route.ts` (EMR), `app/api/events/archive/route.ts` (콜드 아카이빙)

### Server Actions (4개)

| Action | 기능 | Phase |
|--------|------|-------|
| `createWellnessEvent` | 이벤트 DB 저장 | Phase 0 |
| `updateFalseAlarmFlag` | 오탐 마킹 | Phase 2 |
| `generateDailyReport` | 일일 리포트 생성 + Resend 발송 | Phase 2 |
| `updateDeviceStatus` | 하트비트 갱신 | Phase 1 |

## 데이터 모델 (Prisma 5개 모델)

> v2.0 대비 7개 → 5개로 축소. Facility 테이블 및 DeadLetterEvent 삭제.

| 모델 | 핵심 필드 | 비고 |
|-----|---------|------|
| `SensorDevice` | id, locationZone, status, calibrationStatus, lastHeartbeatAt | locationZone MVP는 "BEDROOM" 고정 |
| `WellnessEvent` | id, deviceId, eventType, timestamp, confidenceScore, isFalseAlarm, integrityHash | eventType: ACTIVITY_ALERT / WELLNESS_SCORE / EMERGENCY |
| `UserAccount` | id, email, role, notificationPref | role: GUARDIAN / FACILITY_ADMIN (RBAC는 Phase 2) |
| `UserDevice` | userId, deviceId | M:N 조인 테이블 |
| `DailyReport` | id, deviceId, date, sleepScore, bathroomVisitCount, anomalyFlags, statusCode, aiSummary | statusCode: NORMAL / INSUFFICIENT_DATA / SENSOR_ERROR |

## 무료 티어 제약 (Free Tier Constraints)

| 서비스 | 한계 | 우리 예상 사용량 | 여유 |
|-------|------|--------------|------|
| **Vercel Hobby** | 100 GB-hr 서버리스, Cron 1회/일 | ~20 GB-hr | 🟢 충분 |
| **Supabase Free** | DB 500MB, 7일 미사용 시 Pause | ~100MB | 🟢 충분 |
| **Gemini Flash** | 15 RPM, 1,500회/일 | 50대 × 1회 = 50회/일 | 🟢 충분 |
| **Resend Free** | 100 이메일/일, 3,000/월 | ~50회/일 | 🟢 충분 |

### 무료 유지 핵심 위험 (RISK-07)

**Supabase 7일 미가동 시 자동 Pause** → UptimeRobot 또는 GitHub Actions로 일일 핑 필요

## AI 통합 스펙

| 항목 | 내용 |
|-----|------|
| SDK | Vercel AI SDK (`@ai-sdk/google`) |
| 모델 | Google Gemini 1.5 Flash (기본), env 변수로 교체 가능 |
| 호출 | `generateDailyReport` Server Action → Cron 1회/일 |
| 사용 사례 1 | 웰니스 서사 요약: "어머님이 어젯밤 7.5시간 편안히 수면하셨고..." |
| 사용 사례 2 | 이상 징후 해설: "평소보다 화장실에 오래 머무르셨습니다. 안부 연락을 권해드려요." |
| 비용 | 무료 (15 RPM, 1,500회/일 — 50대 기준 50회/일 사용으로 충분) |
