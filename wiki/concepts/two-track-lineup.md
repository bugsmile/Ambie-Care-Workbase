# Two-Track 라인업

> **KSF #3 / GTM Wave 1+2 — 공공과 프리미엄 동시 점유 전략**  
> 출처: [KSF 보고서](../sources/04-ksfs.md), [VPS GTM](../sources/11-vps.md), [가치사슬 분석](../sources/03-value-chain.md), [AOS/DOS](../sources/09-aos-dos.md)

## 전략 개요

단일 제품이 아닌 **두 개의 별개 라인업**으로 서로 다른 세그먼트를 동시에 공략:

| 구분 | Track A — B2G 공공 보급형 | Track B — B2B 요양시설 프리미엄 |
|-----|--------------------------|-------------------------------|
| **주 고객** | 지자체 (정민석 유형) | 요양시설 (B2B 법인) |
| **채널** | 공공 조달 입찰 | 직접 영업 + EMR 연동 |
| **가격** | 보조금 포함 저가 | ₩100k/월/시설 |
| **하드웨어** | 보급형 모션 센서 | UWB 레이더 풀스펙 |
| **핵심 소구** | 관제 효율 + 예산 절감 | 오탐 제로 + Lock-in |
| **GTM 시기** | Wave 1 클로즈드 베타 | Wave 1 병행 |

---

## 왜 Two-Track인가

### 케어벨 벤치마킹 (가치사슬 분석)
케어벨의 성공 요인: **공공 조달로 레퍼런스 확보 → 프리미엄 B2B 확대** 경로  
→ Rooted도 동일한 레퍼런스 루프 설계

### AOS/DOS 근거
- DOS 최상위 세그먼트:
  - **S1 (B2B 요양시설)**: DOS 3.8, 잠재 매출 ₩26B
  - **S4 (B2G 지자체)**: DOS 3.6, 잠재 매출 ₩19.6B
- 두 세그먼트 모두 Q1 (높은 중요도 + 낮은 현재 만족도) → 동시 공략 정당화

---

## Track A: B2G 공공 보급형

### 타겟
- 지자체 복지 담당자 (정민석 페르소나)
- **예산:** 어르신 1인당 ₩3~5만원/월 서비스 단가 목표

### 핵심 가치
- 방문 요양사 대비 **비용 효율**: 인건비 대체가 아닌 보완
- 야간 응급 사각지대 해소 (독거 어르신 전수 모니터링)
- 조달 KPI: 지자체 1~2곳 레퍼런스 → 전국 복지관 확산

### 제품 스펙 (보급형)
- 기본 모션/적외선 센서 (UWB 제외)
- 관제 대시보드 포함 (신호등 UI, FR-08)
- EMR 연동 불포함 (별도 옵션)

---

## Track B: B2B 요양시설 프리미엄

### 타겟
- 민간 요양시설 원장/운영자
- **예산:** ₩100k/월/시설 (Wave 1), 추후 병상 수 기반 확대

### 핵심 가치
- **EMR 자동 연동** → 이중 기록 제거 → Lock-in
- **UWB 풀스펙** → 오탐 0.3회/월 → 간호사 신뢰도
- 90일 데이터 보존 → 법적 분쟁 방어 (장영희 사례)

### Lock-in 구조
케어벨 벤치마킹: 병원 전산망 통합 후 교체 불가  
→ 초기 도입이 곧 **영구 수주** → LTV 극대화

---

## GTM 로드맵

### Wave 1 (클로즈드 베타)
- **대상:** B2B 소형 요양시설 5~10곳 + B2G 지자체 1~2곳
- **목표:** 오탐 검증 + EMR 연동 레퍼런스 확보
- **수익:** ₩100k × 10시설 = ₩1M/월 (검증 단계)

### Wave 2 (오픈 베타)
- **대상:** B2C 보호자 100~200가구 추가
- **가격:** ₩30k/월 + 설치비 ₩49k
- **목표:** B2C 가격 민감도 검증 + 입소문 채널 확인

### Wave 3 (스케일)
- B2B 전국 확산 + B2G 광역 조달 입찰
- 두 트랙 데이터 공유 → AI 모델 공통 개선
