# 데일리 웰니스 리포트

> **FR-05 / AC-4.x — B2C 잔존율 방어의 핵심 기능**  
> 출처: [PRD FR-05](../sources/12-prd.md), [VPS](../sources/11-vps.md), [CJM P5](../sources/08-customer-journey-map.md), [JTBD 인터뷰](../sources/10-jtbd-interview.md)

---

## 정의

매일 아침 **07:30**에 보호자 앱으로 자동 발송되는 전날 밤 건강 요약 리포트.  
어르신이 아무것도 하지 않아도 보호자가 **"오늘 어머니 어떠셨나요?"** 를 앱으로 확인할 수 있는 루틴.

---

## 리포트 구성 항목

| 항목 | 데이터 | 임계값 (경고 조건) |
|-----|-------|----------------|
| **수면 점수** | 수면 시간, 뒤척임 횟수, 기상 시간 | 수면 5시간 미만 / 야간 기상 3회 이상 |
| **화장실 방문 횟수** | 야간 방문 횟수 | 전주 평균 대비 +50% 초과 (AC-2.2) |
| **낙상 감지 상태** | 감지 이벤트 유무 | 제품 목표는 즉시 푸시, MVP는 이메일 알림 |
| **활동 요약** | 기상 후 이동 패턴 | 장기 비활동 5분 이상 (AC-1.3) |
| **건강 추세** | 7일 이동평균 | 이상 패턴 감지 시 경고 배지 |

---

## 제품 맥락: 왜 아침 7:30인가

### 보호자의 하루 루틴
> *"아침에 출근 준비하면서 핸드폰 볼 때, 어머니 안부를 자연스럽게 확인할 수 있으면 좋겠어요."* — 박지수, CJM P5

- 출근 전 스마트폰 확인 시간대 = **07:00~08:30** (리서치 기준)
- 응급 알림(즉시 푸시)과 데일리 리포트(예약 발송) **이중 채널** 구조
- "오늘 이상 없었음" 확인 = 보호자 **안심 루틴** 형성 → 앱 일간 활성 유지

---

## B2C 잔존율 방어 메커니즘

### CJM P5 — 유지 단계 최대 위기
- P5 이탈 원인: "알림이 너무 많다" / "쓸 일이 없어보인다"
- 데일리 리포트는 **알림 피로 없이** 매일 한 번 정해진 시간에 요약 제공
- 오탐 제로 목표(≤0.3회/월)와 결합 → 신뢰 루틴 형성

### 잔존율 지표 (FR-05 목표)
- AC-4.1: 데일리 리포트 열람률 **60% 이상** (30일 기준)
- AC-4.2: 월간 활성 보호자 이탈율 **5% 미만**

---

## 기술 구현

### 데이터 파이프라인
```text
제품 목표: UWB 센서 → 엣지 처리 → 클라우드 집계 → 07:30 스케줄러 → 푸시 알림
MVP v3.0: 모의 데이터 → Supabase 집계 → Vercel Cron → Gemini 요약 → Resend 이메일
```
- 야간 데이터(22:00~07:00) 배치 집계
- 수면 품질 알고리즘: 뒤척임 횟수 + 무호흡 감지 + 기상 횟수 복합 산정
- 화장실 패턴 오차율 10% 미만 목표 (AC-2.1)

### Not-To-Do (VPS 결정)
- "진단" 언어 배제 (의료기기법 R-01 대응)
- DB/API에서 'diagnosis', 'medical' 단어 완전 제거 (NFR-12)
- 리포트는 **"어떠셨나요" 요약**, 진단/처방 언어 금지

## SRS v3.0 구현 상세

**FR-05는 v3.0에서 Should → Must로 격상** (MVP 킬러 기능 선정)

| 구현 요소 | 상세 |
|---------|------|
| Cron 트리거 | Vercel Hobby 1회/일 (시간 제약) |

| 알림 채널 | Resend Free 이메일 (일 100건 제한) |
| 데이터 보존 | Hot 30일 자동 삭제 (MVP) → Wave 2에서 90일 |
| Phase | Phase 2 (3-5주차) |
