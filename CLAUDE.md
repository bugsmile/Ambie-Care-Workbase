# Rooted — 앰비언트 케어 프로젝트 Wiki 스키마

이 문서는 LLM 에이전트(Claude Code)가 `wiki/` 경로를 지속적으로 유지·운영하기 위한 **운영 규약**입니다.

---

## 프로젝트 개요

**Rooted**는 비접촉 AI 앰비언트 홈 안전 솔루션입니다.  
UWB 레이더 기반 비접촉 센서로 낙상·수면·화장실 패턴을 감지하고, AI 필터링으로 오탐을 제거하며, 보호자 앱으로 데일리 리포트를 제공합니다.

- **원시 소스 경로:** `raw/assets/` (절대 수정 금지, 읽기 전용)
- **위키 경로:** `wiki/` (LLM이 소유하고 유지)
- **스키마 파일:** 이 파일 (`CLAUDE.md`)

---

## Wiki 디렉토리 구조

```
wiki/
├── index.md          ← 전체 페이지 목록 (콘텐츠 카탈로그)
├── log.md            ← 수집/조회/린트 연대순 로그
├── sources/          ← 원시 소스 1건당 요약 페이지 1개
├── entities/         ← 기업, 페르소나 등 고유 주체 페이지
└── concepts/         ← 핵심 개념·전략·기술 페이지
```

---

## 파일 명명 규약

- 영문 소문자, 하이픈(-) 구분: `false-alarm.md`, `persona-park-jisu.md`
- 소스 페이지는 원본 파일번호 접두사 유지: `01-porters-forces.md`
- 엔티티 페이지: `persona-{이름}.md`, `company-{이름}.md`
- 컨셉 페이지: `{개념명}.md`

## 프런트매터 (YAML)

모든 위키 페이지는 아래 프런트매터로 시작합니다:

```yaml
---
title: 페이지 제목
type: source | entity | concept
tags: [태그1, 태그2]
source_count: N          # 이 페이지가 참조하는 원본 소스 수
last_updated: YYYY-MM-DD
---
```

## 운영 워크플로우

### 수집 (Ingest)
새 소스를 `raw/assets/`에 추가한 후 이 명령 사용:
> "이 소스를 위키에 수집해줘"

1. 소스 읽기 → 핵심 정보 추출
2. `wiki/sources/` 에 요약 페이지 작성
3. 관련 엔티티/컨셉 페이지 업데이트 또는 신규 생성
4. `wiki/index.md` 업데이트
5. `wiki/log.md` 에 `## [날짜] ingest | 소스제목` 형식으로 추가

### 조회 (Query)
> "~에 대해 알려줘" / "~를 비교해줘"

1. `wiki/index.md` 로 관련 페이지 식별
2. 관련 페이지 읽기 → 인용 포함 답변 종합
3. 가치 있는 답변은 `wiki/concepts/` 에 새 페이지로 저장 고려
4. `wiki/log.md` 에 `## [날짜] query | 질문 요약` 형식으로 추가

### 린트 (Lint)
> "위키 상태 점검해줘"

1. 고아 페이지, 누락 교차 참조, 모순 내용 확인
2. 오래된 주장, 미작성 중요 개념 식별
3. 발견 사항을 `wiki/log.md` 에 기록

## 핵심 원칙

- **원시 소스는 절대 수정 금지.** LLM은 `raw/`를 읽기만 합니다.
- **위키는 LLM이 전적으로 소유.** 사람이 직접 편집하는 경우는 드물어야 합니다.
- **교차 참조 유지.** 엔티티/컨셉 페이지에서 관련 소스·페이지를 `[[링크]]` 또는 `(링크)` 형식으로 연결합니다.
- **모순 플래그.** 소스 간 상충되는 내용은 해당 페이지에 `> ⚠️ 모순:` 블록으로 명시합니다.
- **용어 규칙.** 제품·DB·API에서 'diagnosis', 'medical', 'patient', '노인', '돌봄', '케어' 등 규제 촉발 및 낙인 단어 사용 금지 (PRD NFR-12, §2.3 #2 참조).
